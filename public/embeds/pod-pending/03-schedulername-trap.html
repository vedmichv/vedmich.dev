Sometimes describe shows nothing: no FailedScheduling, no warnings. Check spec.schedulerName. If it names a scheduler that isn't running, nobody ever claims the Pod.

The default scheduler ignores it, the named one does not exist. No scheduler looks, so no event is ever written. Silence.
