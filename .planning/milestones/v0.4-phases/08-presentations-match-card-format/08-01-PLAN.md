```typescript
const presentations = defineCollection({
  loader: glob({ pattern: '**/*.md', base: './src/content/presentations' }),
  schema: z.object({
    title: z.string(),                       // Deck title (required)
    event: z.string(),                       // Conference/context name (required)
    city: z.string().nullable(),             // Event city — nullable for online/course decks (Slurm) per D-08
    date: z.coerce.date(),                   // Deck date (required)
    description: z.string(),                 // Excerpt for card, 15-30 words (required)
    tags: z.array(z.string()),               // Topic tags — required array per D-08, 2-4 per deck
    slides: z.string().url().optional(),     // External Slidev URL
    video: z.string().url().optional(),      // YouTube URL (reserved, unused Phase 8)
    draft: z.boolean().default(false),
  }),
});
```

Then update the export statement on what was line 31 from:

```typescript
export const collections = { blog, speaking };
```

```typescript

```

**Critical schema notes (do NOT deviate):**
- `description` field is REQUIRED (speaking schema does NOT have this field — presentations does, per D-08).
- `city` uses `z.string().nullable()` NOT `z.string()` — decks like Slurm AI for DevOps have no city (per D-08). `.nullable()` is required, not `.optional()` (the field must be present in frontmatter, but its value may be `null`).
- `tags` is required array `z.array(z.string())` — NO `.optional()` per D-08.
- `slides` and `video` use `z.string().url().optional()` — URL format validation at build time. Markdown files MUST omit these fields entirely when empty (Phase 7 Plan 2 convention — `z.string().url()` fails on empty string `""`).
- Do NOT add `rating` or `highlight` fields (those are Speaking-specific per PATTERNS.md line 94).
- Do NOT rename `description` to `excerpt` (ref uses `excerpt`, but our schema keeps `description` per PATTERNS.md line 1063).
  </action>
