| browser DOM → static HTML (success state) | User-supplied input is **NOT** echoed back into the DOM in any state. The success panel contains only static i18n strings. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-10-01 | Tampering (I — Injection) | Contact.astro submit handler | **mitigate** | **HIGH risk — explicit mitigation required.** User-supplied `name` / `email` / `message` fields could contain CRLF (`\r\n`), semicolons, ampersands, or additional `&cc=`/`&bcc=` segments to inject extra mail headers. **Mitigation:** wrap every field value in `encodeURIComponent()` before interpolation. `%0A` / `%0D` / `%26` / `%3B` are percent-encoded and cannot break out of the URL grammar. The executor MUST NOT use string concatenation or template literals without `encodeURIComponent`. Verified by acceptance criterion: `grep -c "encodeURIComponent" src/components/Contact.astro` returns ≥ 3. |

The success panel's content is pre-rendered by Astro at build time from static i18n strings — no runtime DOM text injection is needed. **Mitigation:** the state machine only toggles visibility via `data-state` attribute + CSS; it never constructs text nodes from user input. If this pattern changes later, use `.textContent` (never `.innerHTML`). Verified by acceptance criterion: `grep -c "\.innerHTML" src/components/Contact.astro` returns 0. |

    type ContactState = 'cta' | 'form' | 'success';

    const CONTACT_EMAIL = 'viktor@vedmich.dev';

    const section = document.getElementById('contact');
    const openBtn = section?.querySelector<HTMLButtonElement>('[data-contact-open-form]');
    const form = section?.querySelector<HTMLFormElement>('[data-contact-form]');
    const cancelBtn = section?.querySelector<HTMLButtonElement>('[data-contact-cancel]');
    const closeBtn = section?.querySelector<HTMLButtonElement>('[data-contact-close-success]');
    const nameInput = form?.querySelector<HTMLInputElement>('input[name="name"]');

    if (section && openBtn && form && cancelBtn && closeBtn && nameInput) {
      function setState(next: ContactState): void {
        section!.setAttribute('data-state', next);
        if (next === 'form') {
          // Match SearchPalette focus pattern — 10ms defer so display toggle completes.
          setTimeout(() => nameInput!.focus(), 10);
        }
      }

      openBtn.addEventListener('click', () => setState('form'));
      cancelBtn.addEventListener('click', () => setState('cta'));
      closeBtn.addEventListener('click', () => setState('cta'));

      form.addEventListener('submit', (e) => {
        // Native HTML5 validation runs FIRST. If invalid, the browser blocks
        // submit and this handler never fires (D-11: no novalidate attribute).
        e.preventDefault();

        const fd = new FormData(form!);
        const name = String(fd.get('name') || '').trim();
        const email = String(fd.get('email') || '').trim();
        const message = String(fd.get('message') || '');

        const subject = name
          ? `vedmich.dev — ${name}`
          : 'vedmich.dev — Contact';
        const body = `From: ${name} <${email}>\n\n${message}`;

        // T-10-01 mitigation: encodeURIComponent on EVERY interpolated value.
        // Prevents CRLF / & / ; injection into mail headers.
        const mailto =
          `mailto:${CONTACT_EMAIL}` +
          `?subject=${encodeURIComponent(subject)}` +
          `&body=${encodeURIComponent(body)}`;

        window.location.href = mailto;
        setState('success');
      });

      // D-34: ESC scoped to the section, NOT document — avoids conflict with
      // SearchPalette's global ESC handler.
      section.addEventListener('keydown', (e) => {
        if (e.key !== 'Escape') return;
        const state = section!.getAttribute('data-state');
        if (state === 'form' || state === 'success') {
          e.preventDefault();
          setState('cta');
        }
      });
    }
  </script>
