<objective>
Ship the three child primitives — VvNode, VvWire, VvPacket — as **register-only** components and update VvStage + _vv-registry.ts to support the register-only architecture. Each child writes its configuration into the active VvStage registry and emits NO visible markup itself; VvStage reads the complete registry after slot-render returns, then composes the final HTML (node divs in .vv-stage-canvas) and SVG (wire paths in <defs>, wire <use>s in one <g>, packet <circle>s in another <g>). This centralizes emission in one place (VvStage) and eliminates the "HTML div nested inside SVG" anti-pattern that plagues naive slot composition.

Purpose: PRIMS-02, PRIMS-03, PRIMS-04. Together with VvStage (Plan 03), this plan completes the composable primitive set so Plan 05 (PodLifecycle refactor) can consume them.
Output: Three new `.astro` files under `src/components/visuals/`, plus updates to `_vv-registry.ts` (PacketRecord + enriched NodeRecord) and `VvStage.astro` (read nodes/packets from registry). After this plan, `<VvStage><VvNode.../><VvWire.../><VvPacket.../></VvStage>` renders valid HTML+SVG end-to-end.

Architecture note: This plan serialized after Plan 03 because it edits Plan 03's artifact (VvStage.astro) and Plan 02's artifact (_vv-registry.ts). Serializing keeps file ownership clean for the orchestrator (no write-write conflicts within a wave).
</objective>
