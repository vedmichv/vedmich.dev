⚠️ DISCOVERY: The pre-refactor wires are NOT mathematically pure edge-point computations — they use the MIDPOINT of the facing edge (which is visually pleasant and what Slidev-style diagrams do). The `_vv-geom.computeEdgePoints` algorithm (Plan 02) produces mathematically correct ray-to-rect intersections, which will differ from the pre-refactor endpoints by up to ~70 pixels on diagonal wires.

IMPLICATION FOR PIXEL PARITY: A diff of ~70 px × ~5 wires × 3 pixel stroke width = ~1000+ anti-aliased pixels difference per wire — exceeds the 150 px tolerance. Two options:

Option A (defer math purity): Add a VvWire `fromPoint` + `toPoint` raw-coordinate override prop that bypasses edge-point math. PodLifecycle refactor uses these overrides to match pre-refactor endpoints exactly. Clean primitives keep the edge-point math as the DEFAULT; authors can override when they need a specific look.

Option B (accept the math): Compute edge-points mathematically; regenerate the baseline screenshot from THIS REFACTORED version, making the POST-refactor snapshot the "new truth". Gives up Wave 0's Pitfall-H protection — cannot prove the refactor changed nothing, only that the new math is consistent.

Option C (midpoint algorithm tweak): Modify `_vv-geom.ts` (Plan 02) to use facing-edge MIDPOINTS for horizontal/vertical layouts (when the ray angle is below some threshold), falling back to ray-intersection for diagonals. More work in Plan 02 but keeps APIs simple.

DECISION: **Option A** — this plan adds a `fromPoint`/`toPoint` prop pair to VvWire as an escape hatch, used by the PodLifecycle refactor for wires where pre-refactor used midpoints instead of ray-intersections. Documented as a rare escape hatch in README (Plan 06). Cheapest path to pixel parity without reworking Plan 02. Locks in Option A by updating VvWire.astro + _vv-registry.ts + VvStage.astro in Task 1 below.

      const fromNode = registry.getNode(w.from);
      const toNode = registry.getNode(w.to);
      if (!fromNode || !toNode) {
        const known = [...registry.nodes.keys()].join(', ') || '(none registered)';
        throw new Error(
          `<VvWire from="${w.from}" to="${w.to}"> — unknown node id. ` +
          `Known node ids: ${known}.`,
        );
      }
      const fromRect: Rect = {
        cx: fromNode.x + fromNode.w / 2,
        cy: fromNode.y + fromNode.h / 2,
        w: fromNode.w, h: fromNode.h,
      };
      const toRect: Rect = {
        cx: toNode.x + toNode.w / 2,
        cy: toNode.y + toNode.h / 2,
        w: toNode.w, h: toNode.h,
      };

      // Escape hatch: if fromPoint/toPoint provided, build path manually; else use computeWirePath
      let d: string;
      if (w.fromPoint || w.toPoint) {
        // Derive missing endpoint via computeEdgePoints so either override still works
        const mathed = computeEdgePoints(fromRect, toRect);
        const [ax, ay] = w.fromPoint ?? [mathed.a.x, mathed.a.y];
        const [bx, by] = w.toPoint ?? [mathed.b.x, mathed.b.y];
        if (w.via && w.via.length > 0) {
          const segments = [`M ${ax} ${ay}`, ...w.via.map(([x, y]) => `L ${x} ${y}`), `L ${bx} ${by}`];
          d = segments.join(' ');
        } else {
          d = `M ${ax} ${ay} L ${bx} ${by}`;
        }
      } else {
        d = computeWirePath(fromRect, toRect, { curve: w.curve, via: w.via });
      }

      wireDefs.push({
        id: w.id,
        d,
        dashed: w.dashed ?? false,
        tone: w.tone ?? 'teal',
        delay: w.delay ?? 0,
      });
    }
