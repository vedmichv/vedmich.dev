## Content Collection Schema

### Collection Name

`presentations` — registered in `src/content.config.ts` alongside `blog` and `speaking` collections.

### Directory Structure

```
src/content/presentations/
├── en/
│   ├── karpenter-prod.md
│   ├── mcp-platform.md
│   ├── slurm-prompt-engineering.md
│   ├── slurm-ai-demo.md
│   ├── eks-multi-az.md
│   └── dkt-workflow.md
└── ru/
    ├── karpenter-prod.md
    ├── mcp-platform.md
    ├── slurm-prompt-engineering.md
    ├── slurm-ai-demo.md
    ├── eks-multi-az.md
    └── dkt-workflow.md
```

**One file per deck.** EN and RU files have matching slugs. 6 decks × 2 locales = 12 markdown files (D-10).

### Frontmatter Schema (Zod)

```ts
z.object({
  title: z.string(),                       // Deck title
  event: z.string(),                       // Conference/context name
  city: z.string().nullable(),             // Event city or null for online/course
  date: z.coerce.date(),                   // Deck date
  description: z.string(),                 // Excerpt for card, 15-30 words
  tags: z.array(z.string()),               // Topic tags (required array)
  slides: z.string().url().optional(),     // External Slidev URL
  video: z.string().url().optional(),      // YouTube URL (reserved, not used Phase 8)
  draft: z.boolean().default(false),
})
```

**Field notes (per D-08):**
- `city` is **nullable** (required field but accepts `null` value) — online/course-based decks like Slurm have `city: null`. This differs from Speaking where `city` is always a string. Render as empty string when null (omit from overline).
- `tags` is **required array** (no `.optional()`) — same as Speaking schema per Phase 7 decision. All decks must have at least one tag.
- `slides` field stores external Slidev URL (e.g. `https://s.vedmich.dev/karpenter-prod/`). When Unified Slides phase ships (future), this becomes optional (deck content lives locally). For Phase 8 migration, populate from current `slug` field via template `https://s.vedmich.dev/{slug}/`.
- `video` field reserved for future YouTube embeds — omit from frontmatter during Phase 8 migration (prevents z.string().url().optional() validation failure on empty strings, same pattern as Phase 7 Plan 2).

### Body Text

Markdown body text = optional long-form notes / key takeaways. Empty string allowed — Phase 8 migrates existing `description` only, no new body content. Body could be populated later from Obsidian vault via QMD search (deferred to future content pass).

## Interaction Patterns

### Homepage Presentations Section

| Interaction | Behavior |
|-------------|----------|
| Card click (anywhere) | Navigate to `https://s.vedmich.dev/{slug}/` in new tab (`target="_blank"`, `rel="noopener noreferrer"`) — external deck link. Whole-card `<a>` anchor pattern (D-23). |
| Card hover | `.card-glow` effect (teal glow on border), title color changes from `text-text-primary` → `text-brand-primary`, slug URL changes from `text-brand-primary` → `text-brand-primary` (no change, already teal), transition 200ms. |
| Tag badge click | None — whole-card anchor captures click. Tags are display labels, not filters (no search/filter in Phase 8). |
| "All decks →" header link | Navigate to `/{locale}/presentations` (internal index page) — same window (D-04, D-05). |
| Scroll animation | Each card fades in via `.animate-on-scroll` (existing global CSS class, IntersectionObserver in `BaseLayout.astro`). |

**Whole-card anchor rationale (D-23, D-24):**
- Phase 8: external link to `s.vedmich.dev/{slug}/` (Slidev deployment)
- Future (Unified Slides v0.5): change to internal `/{locale}/presentations/{slug}` without `target="_blank"`
- Non-breaking data-shape-wise — `slides` frontmatter field already captures external URL

### Index Page `/{locale}/presentations`

| Interaction | Behavior |
|-------------|----------|
| Card click | Same as homepage — external deck link |
| Back link | Navigate to `/{locale}/` (homepage) |
| All other interactions | Same as homepage section |

**Pagination:** None in Phase 8 — all decks on one page (current 6, may grow). Deferred to future phase when deck count > 15-20.

**Accessibility:**
- All external links have `aria-label` with " (opens in new tab)" suffix OR text suffix " →" indicating external navigation
- Tag badges have `role="list"` wrapper, `role="listitem"` per badge (match Speaking pattern Phase 7)
- Whole-card anchor has accessible name from card title (no nested interactive elements)

### Homepage Presentations Section (reference `app.jsx:522-551`)

**Container:**
- `max-w-[1120px] mx-auto` — consistent with all site sections (D-18)

**Card structure (5-row, per D-22):**

```
┌─────────────────────────────────────────────┐
│ 2026-04-19 · AWS Community Day · Bratislava │ ← overline (mono xs muted mb-2)

│                                             │
│ [Kubernetes] [AWS] [Karpenter]              │ ← tags (teal Badge gap-1.5)
└─────────────────────────────────────────────┘
```

**Card:**
- Padding: `p-6` (24px) — reference `app.jsx` uses `padding: 24` in Card primitive (D-21)
- Border radius: `rounded-xl` (16px per `--radius-xl`)
- Background: `bg-surface` (#1E293B per D-17)
- Border: `border border-border` (#334155)
- Hover: `.card-glow` (existing CSS — adds teal glow to border, brightness bump)
- Animation: `.animate-on-scroll` (existing CSS — fade in on scroll)
- Anchor: `<a>` wrapping entire card (D-23)

**5-row details:**

1. **Overline:** `font-mono text-xs text-text-muted mb-2` → renders `{date} · {event}{city ? ' · ' + city : ''}`. Conditional city per D-08 (null for online/course decks). Reference `app.jsx:528-530`.
2. **Title:** `font-display text-lg font-semibold leading-snug mt-0 group-hover:text-brand-primary transition-colors` → 18px/600/line-height 1.3 per reference `app.jsx:531-533`. `mt-0` (no extra top since overline has mb).
3. **Description:** `font-body text-sm text-text-secondary my-2.5 leading-relaxed flex-1` → 14px per reference `app.jsx:535-537`. **my-2.5 = 10px** (ref `margin: '10px 0'`) — documented spacing exception. `flex-1` pushes URL+tags to bottom.
4. **Slug URL:** `font-mono text-[11px] text-brand-primary tracking-[0.06em] mt-2 mb-2.5 truncate` → 11px SOLID teal (no /80 alpha, D-22), tracking 0.06em per ref `app.jsx:539-541`. Rendered as `s.vedmich.dev/{slug}`. mt-2/mb-2.5 (ref `marginTop: 8` `marginBottom: 10`).
5. **Tags:** `flex flex-wrap gap-1.5` — 6px gap per reference `app.jsx:543`. Badges per D-01 Badge style (11px/600/0.08em uppercase mono, teal bg/border/text).

**Header row (section title + "All decks →" link):**
- `flex items-baseline justify-between gap-6 mb-2` — title left, link right, aligned to baseline
- Link style: `text-sm text-text-muted hover:text-brand-primary transition-colors whitespace-nowrap` (D-04)

**Responsive:**
- Mobile (< 640px): 1-col grid, cards full-width stack
- Tablet (640-1024px): 2-col grid
- Desktop (≥ 1024px): 3-col grid

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Semantic HTML | Cards in `<div>` grid (not `<ul>` — cards are content blocks, not list items). Tags within card use `<div role="list">` wrapper with `<span role="listitem">` per badge. |
| Keyboard navigation | All interactive elements (cards, header link, back link) focusable via Tab, visible focus ring (teal 2px offset) via `focus-visible:ring-2 ring-brand-primary ring-offset-2 ring-offset-bg-base`. |
| Screen reader | External link suffix " →" in "All decks →" indicates navigation. Whole-card anchor accessible name = card title (no nested interactive elements). Tag list semantics via role attributes. |
| Color contrast | All text meets WCAG AA — text-primary 13.5:1, text-secondary 7.8:1, text-muted 5.3:1, teal accent 4.8:1 against `--bg-surface`. |
| Focus visible | Tailwind default focus-visible ring on all links/buttons. |

**External link accessibility:**
- Whole-card anchor has `target="_blank"` and `rel="noopener noreferrer"`
- `aria-label` on card anchor: `{deck title} (opens in new tab)` OR rely on browser default behavior (user agent announces "link, opens in new window")
- "All decks →" link does NOT have `target="_blank"` (internal navigation)

## Data Migration Plan

### Source

`src/data/social.ts` — `presentations` array (6 decks, lines 52-113).

### Target

`src/content/presentations/{en,ru}/*.md` — one markdown file per deck with frontmatter.

### Migration Steps

1. Extract each deck from `presentations` array
2. Create EN markdown file: `src/content/presentations/en/{slug}.md` (slug = existing `slug` field)
3. Create RU markdown file: `src/content/presentations/ru/{slug}.md` (mirror)
4. Populate frontmatter:
   - `title` ← `title`
   - `event` ← extract event name from `event` field (before ` · ` separator if city present)
   - `city` ← extract city from `event` field (after ` · ` separator) OR `null` if no separator (e.g. "Slurm AI for DevOps" → `city: null`)
   - `date` ← `date` (ISO string → coerce.date)
   - `description` ← `description` (ref calls this `excerpt`, our schema calls it `description`)
   - `tags` ← `tags` (array, preserve Mixed case as-is)
   - `slides` ← construct from slug: `https://s.vedmich.dev/{slug}/`
   - `video` ← omit (reserved for future, not used Phase 8)
   - `draft` ← `false` (all 6 decks are live)
5. Body text: empty string initially (executor can expand later via QMD vault search if desired, or leave empty)
6. After migration complete, remove `export const presentations` from `social.ts` per D-25 (clean break, no backward-compat shim)
7. Update `src/data/search-index.ts` to query Content Collection instead of importing from `social.ts` per D-26

**Executor decision:** Body text content during migration vs defer to future content pass. Default: empty body, migrate verbatim from data structure.

**Filename convention (D-10):** `{slug}.md` (NOT `YYYY-slug.md` like Speaking). Speaking uses year-prefix for chronological sorting in file tree; Presentations uses slug directly since they're all recent (2025-2026) and slug is already unique.
