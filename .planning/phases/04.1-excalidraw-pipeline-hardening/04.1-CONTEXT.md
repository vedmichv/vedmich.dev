### Security Hardening (Plan candidate: 04.1-01)

- **D-02 (SEC-C01 — symlink bypass):** Replace `validatePath` with a realpath-based check that walks up to the first existing ancestor, `fs.realpathSync` that ancestor, reattaches the tail, and enforces `startsWith(REPO_ROOT + path.sep) || === REPO_ROOT`. Drop `TMP_ROOT` from the write allowlist entirely — tests move from `os.tmpdir()` writes to a repo-local `tests/tmp/` directory that is gitignored. Add a symlink-bypass regression test (create symlink pointing at `/tmp/victim.txt`, attempt write, assert script exits non-zero and victim file is unchanged).
- **D-03 (SEC-C02 — fictional SVGO `removeScripts`):** Extend `SVGO_CONFIG.plugins` with explicit `removeScripts` + `removeAttributesBySelector` entries appended after `preset-default`. Target selectors: `[onload], [onclick], [onmouseover], [onerror], [onfocus]` plus `foreignObject, iframe, script`. Verify against a malicious fixture (`<svg><script>alert(1)</script><foreignObject><iframe src="javascript:alert(1)"/></foreignObject></svg>`) — all three elements removed by SVGO post-pass. Update Phase 4's T-04-02 threat-model claim to reflect that explicit plugins, not `preset-default`, are the defense.
- **D-04 (SEC-H03 — SSRF via dataURL):** In `validateFilesBlob`, extend validation: for each `diagram.files[*]`, if `dataURL` is a string, reject unless it matches `^data:image/(png|jpeg|gif);base64,`. `file://`, `http(s)://`, `data:image/svg+xml`, and raw non-base64 `data:` URIs are rejected. Tests: 4 negative fixtures (each rejected scheme) + 1 positive (valid `image/png;base64,`) + boundary test on the 100 KB blob size guard.
- **D-05 (SEC-H04 — parser-error silent success):** After `excalidrawToSvg(diagram)`, inspect `svgEl.outerHTML` — if it starts with `<parsererror` or contains `<parsererror ` as a root-ish child, throw with a diagnostic error naming the source path. Tests: malformed-diagram fixture that trips the jsdom parser but currently ships a 116-byte `<parsererror>` SVG.
- **D-06 (Q-05 — `ALLOWED_WRITE_ROOTS` refactor):** Refactor the hardcoded `REPO_ROOT` + `TMP_ROOT` checks into an `ALLOWED_WRITE_ROOTS` array with a single loop. After D-02 drops `TMP_ROOT`, the array is `[REPO_ROOT]` — the refactor still lands because it makes future allowlist additions safe and simplifies the error message ("resolves outside all allowed roots: [REPO_ROOT]").

### Visual / UX Hardening (Plan candidate: 04.1-02)

- **D-07 (UX-01 white background — SVGO post-process):** Extend `SVGO_CONFIG.plugins` with `removeAttributesBySelector` pattern `path[fill="#fff"][d^="M0 0h"]` that matches only full-canvas leading white rectangles (the `M0 0h<width>v<height>` path form Excalidraw emits for `viewBackgroundColor: "#ffffff"`). Narrow selector is intentional — legitimate white shapes inside the diagram (text backgrounds, callouts) have different `d` prefixes and survive. Test: fixture with both a leading `M0 0h...` white rect AND an interior `M100 50...` white shape — assert only the rect is removed.
- **D-08 (UX-04 Helvetica fallback chain):** Script adds a post-SVGO string replace (or SVGO `addAttributesToSVGElement`-style override if cleaner): `font-family="Helvetica"` → `font-family="Helvetica, Arial, sans-serif"` on every `<text>` element. Windows (no Helvetica installed) falls back to Arial — metrics close enough that existing label-box geometry holds. Future Helvetica diagrams inherit automatically. Budget impact: +24 bytes per diagram worst case, well under 10 KB ceiling. Re-export karpenter SVG to pick up the change.
- **D-09 (UX-02/UX-03 MCP re-author):** Re-author `diagrams-source/2026-03-02-mcp-servers-plainly-explained/client-server.excalidraw.json` in Excalidraw desktop/web: (a) bump every label `font-size` from 16 to 28-32 px so that `28 * (343/820) = 11.7 px` on 375 px mobile stays above the 9 px legibility floor; (b) expand viewBox from 820×100 to 820×300 for ~2.7:1 aspect ratio with vertical breathing room — reads as intentional illustration, not banner separator; (c) set `viewBackgroundColor: null` for transparent canvas (belt-and-suspenders with D-07's script-level strip). Update MCP MDX `<img width height>` to new intrinsic dimensions. Karpenter diagram NOT re-authored — its 1020×520 aspect already reads correctly (1.96:1) and its 16 px labels scale to `16 * (343/1020) = 5.4 px` on mobile BUT the diagram is loading="lazy" below the fold and wider aspect makes mobile readers pinch-zoom anyway; accept as known limitation (captured as D-09b below).
- **D-09b (karpenter mobile label scaling — accepted):** Karpenter split-ownership diagram labels scale to ~5.4 px on 375 px mobile. Re-author to 28-32 px would require a third round-trip into Excalidraw and risks busting the 10 KB budget at 17 elements. Accept for v1.0; captured as a follow-up in Deferred Ideas below. `<figcaption>` content (D-10) provides the text-equivalent for mobile readers.
- **D-10 (UX-06 figure/figcaption a11y pattern):** Wrap both shipped `<img>` tags in `<figure class="my-6">...<figcaption class="text-sm text-text-muted mt-2 text-center">caption</figcaption></figure>`. Captions are bilingual — EN post has EN caption, RU post has RU caption. Content: 1-2 sentence plain-language description of the diagram (equivalent to the SVG `<desc>` but readable in the DOM for all users). Zero new i18n JSON keys needed (caption is per-post inline content, not reusable).

### Code Quality + Test Coverage (Plan candidate: 04.1-03)

- **D-11 (Q-02 JSON.parse error UX):** Add `parseJsonOrThrow(text, pathForErr)` helper that wraps `JSON.parse`, catches `SyntaxError`, throws `new Error('failed to parse JSON in ' + path.relative(REPO_ROOT, pathForErr) + ': ' + originalError.message)`. Replace both `JSON.parse` call sites (meta + source). Test: malformed-meta fixture + malformed-source fixture both assert error names the offending file.
- **D-12 (Q-03/Q-04 meta validation):** Extend meta validation beyond current "must have `title` + `descEn`". Required: `typeof meta.title === 'string' && meta.title.length > 0`, `typeof meta.descEn === 'string' && meta.descEn.length > 0`. Optional-but-typed: if `meta.descRu` present, must be string. If `meta.title` is an object (`{en: "..."}` — common author mistake), error names the key and suggests the flat-string shape. Test: 3 negative fixtures (`title: ""`, `title: { en: "x" }`, `title: null`) — each asserts descriptive error.
- **D-13 (Q-08 FILES_BLOB_BUDGET test):** Add fixture `oversize-files.excalidraw.json` with `files: { "fake-id": { dataURL: "data:image/png;base64,AAAA..." (>100 KB) } }` and assert `validateFilesBlob` rejects with non-zero exit. Pair-tests D-04's dataURL-scheme guard (positive-valid scheme but too big → rejected on size; invalid scheme → rejected on scheme).
- **D-14 (Q-01 idempotency — doc caveat + test):** Add test `assert byte-identical SVG on 2 runs of minimal.excalidraw.json` — passes today because fixture has hardcoded seeds. Add "Determinism caveat" H3 to runbook Gotchas section: "Re-exports from Excalidraw UI regenerate `seed`/`versionNonce` on every edit session. Expect SVG `git diff` to show byte churn even when the visual is identical. Commit workflow: edit → commit `.excalidraw.json` first → run pipeline → commit SVG. Don't hand-touch seeds; they drive the rough.js jitter." Update the `// Re-runnable` comment in the script header to clarify: "Re-runnable for byte-identical output ONLY when `.excalidraw.json` is stable; Excalidraw UI re-exports regenerate seeds and will produce visually-identical but byte-different SVGs." Rejected alternative: stripping/normalizing seeds in the script — changes the hand-sketched aesthetic (deterministic jitter pattern per element.id instead of pseudo-random) and adds ~20 LOC of walker code.
- **D-15 (Q-12 escapeXml Unicode):** Parametric tests for the existing `escapeXml` covering: emoji surrogate pair (codepoint U+1F680 rocket), RTL mark (U+200F), zero-width joiner (U+200D), combining diacritic (letter `e` followed by U+0301), and CJK ideographs (U+4E2D U+6587). All should pass through unchanged (not in the XML-reserved set `<>&"'`) — tests exist to pin behavior against future helper rewrites. Planner MUST encode these codepoints as JS `\u{...}` or `\u....` escape sequences in the test source file; never paste raw invisible bytes into the fixture or the test (keeps the test file reviewable in diffs and trips zero injection-scanner heuristics).
- **D-16 (Q-10 test style):** Replace `test.after(() => fs.unlinkSync(dest))` patterns with `try { ... } finally { unlinkSync }` — the node:test `after` hook fires after all tests in a describe, not after each test, and can miss cleanup on skipped paths. Style-only polish.
- **D-17 (Q-13 optional `--quiet` flag):** Add a `--quiet` flag that suppresses the success `✓` + `intrinsic:` + `title:` + `desc:` log lines but keeps errors on stderr. Useful for batch loops. Does NOT change exit code behavior. Small enough (~6 LOC) that it lands in the same plan.
