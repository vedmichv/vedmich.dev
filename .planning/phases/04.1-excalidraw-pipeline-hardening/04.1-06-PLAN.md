    ## SEC-L02 closure note (D-23)

    SEC-L02 is the latent regex-injection finding from the 04-REVIEW BL-02. The BL-02 correctness-time-bomb exploit vector required a wrapper-side change that let `>` through into an attribute value, which would then trip the injection regex `/<svg([^>]*)>/` at scripts/excalidraw-to-svg.mjs line 129. That path was latent today (wrapper's XMLSerializer normalizes `>` to `&gt;` in all attribute values tested in SECURITY-REVIEW.md).

    D-03 (ship 04.1-01) adds explicit `'removeScripts'` + `removeAttributesBySelector` for 5 event-handler attributes as SVGO post-pass plugins. ANY surviving `<script>` element or `onload=`/`onclick=`/etc. attribute — regardless of what path it took to get into the SVG (wrapper regression, BL-02 regex split gone wrong, custom plugin that adds raw HTML) — is stripped before the SVG is written to disk. The defense-in-depth is now real, not claimed.

    **Closure:** SEC-L02 marked closed in this SUMMARY as a side-effect of D-03. No separate fix needed in 04.1-06.
