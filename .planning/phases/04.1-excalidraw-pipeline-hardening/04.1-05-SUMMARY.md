### Auto-fixed Issues

**1. [Rule 1 - Bug] Edit tool normalized JS escape sequences to raw UTF-8 bytes in the Task 3 test file**
- **Found during:** Task 3 post-Edit verification (`grep -qP '[\x{200B}-\x{200F}\x{202A}-\x{202E}\x{2060}-\x{206F}]' tests/unit/excalidraw-to-svg.test.ts` returned exit 0 with 2 matches — indicating raw invisible bytes landed in the test source despite D-15's explicit ESCAPE-SEQUENCES-ONLY mandate).
- **Issue:** The Edit tool's `new_string` parameter silently converted 4 of the 5 Unicode codepoints from their JS escape-sequence form to raw UTF-8 bytes on disk. Specifically: `'\u200F'` → `'\u200F'` (bytes `e2 80 8f`), `'\u200D'` → `'\u200D'` (bytes `e2 80 8d`), `'e\u0301'` → `'é'` in decomposed form (bytes `65 cc 81`), and `'\u4E2D\u6587'` → `'\u4E2D\u6587'` (bytes `e4 b8 ad e6 96 87`). Only `\u{1F680}` (the 6-character long-form escape) survived Edit's normalization unchanged. The U+200F and U+200D bytes would trip the raw-invisible-byte detector; U+0301 combining mark and U+4E2D CJK would not trip the range-based detector but would still violate D-15's "never raw bytes" mandate by making the diff less reviewable and risking editor-normalization during future edits.
- **Fix:** Byte-level Python substitution `python3 << PYEOF ... data.replace(<raw-utf8-bytes>, <ascii-backslash-u-form>) ... PYEOF` applied post-Edit to restore all 4 broken codepoints to their escape-sequence form. Each substitution validated with a count assertion (each old pattern matched exactly 1 time). The acute case required a separate substitution because the first attempt targeted the precomposed `é` (U+00E9) but Edit had written the decomposed form e + U+0301 (which was what the plan wanted anyway — the second Python pass replaced `65 cc 81` → `e\u0301` as ASCII text).
- **Files modified:** `tests/unit/excalidraw-to-svg.test.ts` — 4 string-literal surgical replacements, ~0 LOC net change (bytes substituted in place).
- **Verification:** Post-fix: `grep -qP '[\x{200B}-\x{200F}\x{202A}-\x{202E}\x{2060}-\x{206F}]' tests/unit/excalidraw-to-svg.test.ts` → exit 1 (zero matches).
