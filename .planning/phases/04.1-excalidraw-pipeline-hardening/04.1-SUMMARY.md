# Phase 04.1 — Excalidraw Pipeline Hardening — SHIPPED

## Executive summary

Phase 04.1 remediated the 58 post-phase review findings (2 CRITICAL + 2 HIGH security, 4 blocking UX, 7 code-quality, 13 docs-drift, 1 SEC-LOW, + P3/P4 polish) logged against the Phase 4 Excalidraw shipment in `reviews/CONSOLIDATED.md`. All P1 CRITICAL and HIGH findings (SEC-C01 symlink bypass, SEC-C02 fictional SVGO strip, SEC-H03 SSRF via dataURL, SEC-H04 parsererror silent success) are now closed by script-level hardenings in `scripts/excalidraw-to-svg.mjs`. Pipeline UX fixes (leading white-rect strip + Helvetica→fallback-chain) ship retroactively through re-export of the 2 live diagrams. All 4 MDX `<img>` tags wrapped in `<figure>/<figcaption>` with bilingual captions. Docs drift swept across 6 files.

Pipeline is now production-safe for untrusted PR contributions (attacker-planted symlinks, malicious SVG content, SSRF-bait dataURLs all refused at the gate). Shipped SVGs remain clean; the hardenings forward-close exploit paths without user-visible churn beyond the bilingual figcaptions and the MCP re-author (Excalifont label size 16→28 px).

## SEC-L02 closure note (D-23)

SEC-L02 is the latent regex-injection finding logged against BL-02 in the 04-REVIEW. The exploit vector required a wrapper-side change that let `>` through into an attribute value, which would then trip the injection regex `/<svg([^>]*)>/` in `scripts/excalidraw-to-svg.mjs`. That path was latent today — the wrapper's XMLSerializer normalises `>` to `&gt;` in all attribute values tested in `reviews/SECURITY-REVIEW.md`.

D-03 (shipped in 04.1-01) adds explicit `'removeScripts'` + `removeAttributesBySelector` for 5 event-handler attributes as SVGO post-pass plugins. ANY surviving `<script>` element or `onload=`/`onclick=`/etc. attribute — regardless of what path it took to get into the SVG (wrapper regression, BL-02 regex split gone wrong, custom plugin that adds raw HTML) — is stripped before the SVG is written to disk. The defense-in-depth is now real, not claimed.

**Closure:** SEC-L02 marked closed as a side-effect of D-03. No separate fix needed in 04.1-06.

## Forced-colors verdict (D-24) — DEFER'd

D-24 was a human-verified Playwright smoke check on 4 URLs × 2 viewports × 2 emulation modes (default + forced-colors + prefers-contrast:more) = 16 screenshots. The check was NOT exercised in 04.1-06 because:

1. The inherited Phase 4 hard-coded `stroke="#1e1e1e"` in both shipped SVGs is NOT regressed by Phase 04.1. Both SVGs re-exported via the hardened pipeline retain the same stroke color (re-export is byte-different but visually-identical).
2. D-07 white-rect strip arguably HELPS forced-colors (the bright slab no longer competes with the system color palette).
3. The full remediation (per-diagram SVG token swap to `currentColor` + transparent fill) is out of scope for v1.0 per D-03b Phase 4 deferral.

Captured in §Deferred for v1.1. Flag if any low-vision reader surfaces feedback.
