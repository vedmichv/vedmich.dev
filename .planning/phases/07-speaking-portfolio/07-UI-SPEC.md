## Content Collection Schema

### Collection Name

`speaking` — registered in `src/content.config.ts` alongside `blog`.

### Directory Structure

```
src/content/speaking/
├── en/
│   ├── 2026-karpenter-cost-saving.md
│   ├── 2024-kubernetes-autoscaling.md
│   ├── 2024-gitops-best-practices.md
│   ├── 2023-reinvent-eks-anywhere.md
│   └── ...
└── ru/
    ├── 2026-karpenter-cost-saving.md
    ├── 2024-kubernetes-autoscaling.md
    └── ...
```

**One file per talk.** EN and RU files have matching slugs.

### Frontmatter Schema (Zod)

```ts
z.object({
  title: z.string(),                    // Talk title
  event: z.string(),                    // Conference/event name
  city: z.string(),                     // Event city
  date: z.coerce.date(),                // Event date
  tags: z.array(z.string()),            // Topic tags (lowercase)
  video: z.string().url().optional(),   // YouTube URL
  slides: z.string().url().optional(),  // Slidev URL (s.vedmich.dev/slug/)
  rating: z.string().optional(),        // Speaker rating e.g. "4.7/5.0"
  highlight: z.string().optional(),     // Highlight text e.g. "Speaker rating: 4.7/5.0"
  draft: z.boolean().default(false),
})
```

### Body Text

Markdown body text = talk description, key takeaways, notes. Optional — some talks may have empty body initially during migration.

## Interaction Patterns

### Homepage Speaking Section

| Interaction | Behavior |
|-------------|----------|
| Year text click | None — year is display text, not interactive |
| Event name click | None — event is display text, not interactive |
| Talk title click | Navigate to `/{locale}/speaking/{slug}` |
| `→` arrow | Part of talk text, not separate clickable element |
| "View all →" link | Navigate to `/{locale}/speaking/` (full portfolio page) |

**Scroll animation:** Each year group fades in via `.animate-on-scroll` (existing global CSS class, IntersectionObserver in `BaseLayout.astro`).

### Individual Talk Page

| Interaction | Behavior |
|-------------|----------|
| "Watch on YouTube" button | External link, `target="_blank"`, `rel="noopener noreferrer"`, opens YouTube video in new tab |
| "View Slides" button | External link, `target="_blank"`, `rel="noopener noreferrer"`, opens Slidev deck in new tab |
| Tag badge click | None — tags are display labels, not filters (no search/filter in v0.4) |
| "← Back to Speaking" link | Navigate to `/{locale}/speaking/` |
| YouTube embed | iframe with `loading="lazy"`, aspect ratio 16:9, responsive width |

**Video embed implementation:** Use lite YouTube embed library OR native iframe with lazy loading. Executor chooses based on performance vs complexity trade-off. If lite-youtube, add NPM package and import. If iframe, use `loading="lazy"` attribute.

**Accessibility:**
- All external links have `aria-label` with " (opens in new tab)" suffix
- Video iframe has `title` attribute with talk title
- Tag badges have `role="list"` wrapper, `role="listitem"` per badge

## Layout Specifications

### Homepage Speaking Section (reference `app.jsx:456-486`)

**Container:**
- `max-w-[1120px] mx-auto` — consistent with all site sections
- `py-20 sm:py-28` — 80px mobile, 112px desktop vertical padding
- `px-6` — 24px horizontal padding

**Grid structure per year:**
```
┌─────────────┬──────────────────────────────────────┐
│   2026      │  Event · City                        │
│  (100px)    │    → Talk title 1                    │
│             │    → Talk title 2                    │
│             │  ★ Highlight (if present)            │
│             ├──────────────────────────────────────┤
│             │  Event · City                        │
│             │    → Talk title 3                    │
└─────────────┴──────────────────────────────────────┘
```

- Grid: `grid grid-cols-[100px_1fr] gap-6` per year group
- Year: teal display-36 bold, `-0.02em` letter-spacing
- Events: left border `border-l-2 border-border`, `pl-5` (20px padding)
- Event gap: 16px vertical between events within same year
- Year group gap: 32px vertical between year groups

**No cards, no timeline dots, no vertical line.**

### Individual Talk Page

**Container:**
- `max-w-3xl mx-auto` — narrower than homepage (768px max-width for prose readability)
- `py-20 sm:py-28` — same vertical padding as homepage sections
- `px-6` — 24px horizontal padding

**Page structure:**
1. Back link — top-left, teal, with `←` arrow, `text-sm`
2. h1 — talk title, display-36 bold, `mb-4`
3. Meta row — event · city · date, text-secondary, `mb-6`
4. Tag badges row — teal/amber badges, `mb-8`
5. Video embed (if present) — 16:9 aspect ratio, `mb-8`
6. CTA buttons row — "Watch" amber, "View Slides" teal, `gap-3`, `mb-8`
7. Body prose — markdown content, `prose prose-invert prose-lg`, teal accent links

**Responsive:**
- Mobile (< 640px): full-width video, stacked buttons
- Desktop (≥ 640px): video maintains aspect ratio, buttons inline

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Semantic HTML | Year groups in `<section>`, talks in `<ul>` / `<li>`, headings in correct order |
| Keyboard navigation | All interactive elements focusable via Tab, visible focus ring (teal 2px offset) |
| Screen reader | External link indicators via `aria-label`, video `title`, list semantics for talks |
| Color contrast | All text meets WCAG AA — text-primary 13.5:1, text-muted 5.3:1, teal accent 4.8:1 |
| Focus visible | `focus-visible:ring-2 ring-brand-primary ring-offset-2 ring-offset-bg-base` on all links/buttons |

**No interactive elements rely solely on color** — external link icon + text label, `→` arrow is visual + semantic prefix.

## Data Migration Plan

### Source

`src/data/social.ts` — `speakingEvents` array (5 events, 6 talks total across 2026/2024/2023).

### Target

`src/content/speaking/{en,ru}/*.md` — one markdown file per talk with frontmatter.

### Migration Steps

1. Extract each talk from `speakingEvents` array
2. Create EN markdown file: `src/content/speaking/en/{YYYY-slug}.md`
3. Create RU markdown file: `src/content/speaking/ru/{YYYY-slug}.md` (mirror)
4. Populate frontmatter: title (talk), event (event.name), city (event.location), date (infer from year), tags (infer from title), video (add if known), slides (add if known), rating (event.highlight if present), draft (false)
5. Body text: empty initially (executor can expand later via QMD vault search)
6. After migration complete, remove `speakingEvents` export from `social.ts`

**Executor decision:** How much body text to write during migration vs defer to future content pass.
