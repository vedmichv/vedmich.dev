### `scripts/excalidraw-to-svg.mjs` (NEW — single-purpose ESM CLI, `.excalidraw.json → .svg` with SVGO + a11y injection)

**Analog (exact):** `scripts/generate-icons.mjs`.

**Why this analog:** Same project convention — shebang-less ESM `.mjs` in `scripts/`, top-level `await` in `main()`, stdlib-first imports, one-shot invocation (`node scripts/X.mjs`), `process.exit(1)` on failure, idempotent re-run, output committed to `public/` but not run on every `astro build`.

**Imports block pattern** (lines 1-11 of `scripts/generate-icons.mjs` verbatim — mirror the shape):

```javascript
// @ts-check
// Source: Phase 11 (Logo + favicon refresh) — CONTEXT.md D-09 & D-10
// One-shot asset pipeline. Re-runnable via `node scripts/generate-icons.mjs`.
// Output is committed to public/ + .design-handoff/; NOT run on every build.

import fs from 'node:fs/promises';
import path from 'node:path';
import os from 'node:os';
import { fileURLToPath } from 'node:url';
import sharp from 'sharp';
import pngToIco from 'png-to-ico';
```

**What to mirror for Phase 4:**
- `// @ts-check` on line 1 (enables JSDoc type checking via tsserver — no TS compile step).
- Source comment naming the phase + CONTEXT.md decision (Phase 4, D-02c script contract).
- Import block: `fs from 'node:fs/promises'`, `path from 'node:path'`, `fileURLToPath from 'node:url'`, then the phase-specific deps (`excalidrawToSvg from '@aldinokemal2104/excalidraw-to-svg'`, `{ optimize } from 'svgo'`).
- Drop `os` (Phase 4 doesn't need `os.homedir()` — `REPO_ROOT` is enough).

**REPO_ROOT + top-level constants pattern** (lines 13-16 verbatim):

```javascript
const __filename = fileURLToPath(import.meta.url);
const __dirname = path.dirname(__filename);
const REPO_ROOT = path.resolve(__dirname, '..');
const PUBLIC_DIR = path.join(REPO_ROOT, 'public');
```

**What to mirror:** Identical `__filename`/`__dirname`/`REPO_ROOT` idiom. Phase 4 doesn't need `PUBLIC_DIR` as a constant because the destination path is a CLI arg, not hardcoded — but the `REPO_ROOT` is useful for path-traversal validation (security).

**Budget gate pattern** (lines 83-89 verbatim — this is the 10 KB check Phase 4 mirrors, but after SVGO instead of after sharp encode):

```javascript
const heroBuf = await renderPng(SRC_LOGO_HERO, 64);
if (heroBuf.length > 10 * 1024) {
  throw new Error(
    `vv-logo-hero.png is ${heroBuf.length} B, exceeds 10 KB budget`
  );
}
```

**What to mirror:** Exact same 10 KB guard pattern. Phase 4 D-05 wants the same shape: compute byte length, compare to `10 * 1024`, throw with the byte count printed. **One adjustment per CONTEXT D-05:** Phase 4 should additionally print an SVGO diff summary to stderr before `process.exit(1)` (the guard hint: "Simplify the diagram or zoom-to-fit in Excalidraw before re-exporting."). Use `console.error` for the diagnostic, match `main().catch()` for the exit.

**Logging line pattern** (lines 54-60 verbatim):

```javascript
async function copyFile(src, dest) {
  await fs.copyFile(src, dest);
  console.log(`  copy  ${path.relative(REPO_ROOT, dest)}`);
}

async function writeBuffer(buf, dest) {
  await fs.writeFile(dest, buf);
  console.log(`  write ${path.relative(REPO_ROOT, dest)}  (${buf.length} B)`);
}
```

**What to mirror:** Post-write logging pattern — relative-path + byte count in parens. Phase 4's success log should be similar: `console.log('✓ ' + path.relative(REPO_ROOT, destPath) + ' (' + bytes + ' B)')`. Use the "✓" checkmark for success (CONTEXT RESEARCH.md §Pattern 1 uses it) or drop it to stay closer to `generate-icons.mjs` (which uses no prefix — just `copy` / `write` verbs). Planner picks.

**Error exit pattern** (lines 155-158 verbatim — the canonical tail):

```javascript
main().catch((err) => {
  console.error('[generate-icons] FAILED:', err);
  process.exit(1);
});
```

**What to mirror:** Exact same tail. Replace `[generate-icons]` with `[excalidraw-to-svg]`. Same behavior: any thrown error bubbles to the top-level `catch`, logs to stderr, exits 1.

**What Phase 4 adds (not present in `generate-icons.mjs`):**
1. **CLI arg parsing** (D-02c positional-args contract): `const [, , srcPath, destPath] = process.argv;` at the top of `main()`. No `commander` / `yargs` — matches generate-icons' zero-arg-lib convention.
2. **Meta sibling file read** (D-02d mandatory a11y): `const metaPath = srcPath.replace(/\.excalidraw\.json$/, '.meta.json');` + `const meta = JSON.parse(await fs.readFile(metaPath, 'utf8'));` + validation (`if (!meta.title || !meta.descEn) throw ...`).
3. **SVGO config object** (top-level constant, mirror `WEBMANIFEST` const at lines 39-50). Shape:
   ```javascript
   const SVGO_CONFIG = {
     multipass: true,
     plugins: [{ name: 'preset-default', params: { overrides: { removeDesc: false } } }],
   };
   ```
   Per RESEARCH Pitfall 1, **only** `removeDesc: false` override is needed (v4 removed `removeTitle`/`removeViewBox` from preset-default defaults).
4. **`<title>`+`<desc>` injection** (D-03c dual-channel a11y): string-manipulate SVG output BEFORE SVGO (per RESEARCH Pitfall 5 — post-SVGO injection is a position bug). Pattern:
   ```javascript
   svgString = svgString.replace(
     /<svg([^>]*)>/,
     `<svg$1><title>${escapeXml(meta.title)}</title><desc>${escapeXml(meta.descEn)}</desc>`
   );
   ```
5. **viewBox→width/height parse** (D-03d): regex the SVG for `viewBox="x y w h"`, extract w/h, print to stdout for author to paste into MDX `width`/`height` attrs.
6. **`escapeXml` helper** (6-liner, per RESEARCH "Don't Hand-Roll" table row — exception granted because it's trivially small):
   ```javascript
   function escapeXml(str) {
     return str.replace(/[<>&"']/g, (c) => ({
       '<': '&lt;', '>': '&gt;', '&': '&amp;', '"': '&quot;', "'": '&apos;'
     }[c]));
   }
   ```
7. **mkdir -p for destination** (mirror generate-icons line 89 pattern): `await fs.mkdir(path.dirname(destPath), { recursive: true });` before write.

**Lines NOT to copy from generate-icons:** The whole "mirror to `.design-handoff/`" block (lines 125-151) — Phase 4 output is single-destination (`public/blog-assets/<slug>/diagrams/`). No handoff mirror.

## Shared Patterns

### Node ESM Script Skeleton (mirror `scripts/generate-icons.mjs`)

**Source:** `scripts/generate-icons.mjs`.

**Apply to:** Any new build-time asset pipeline script in this project. Phase 4's target: `scripts/excalidraw-to-svg.mjs`.

**Contract:**
1. `// @ts-check` on line 1 (enables tsserver JSDoc type checking without a TS compile step).
2. Source comment block (lines 2-4): `// Source: Phase X — CONTEXT.md D-YY`, one-line purpose, "Re-runnable via `node scripts/X.mjs`".
3. Imports: `fs from 'node:fs/promises'` + `path from 'node:path'` + `fileURLToPath from 'node:url'` first, then third-party deps.
4. `__filename`/`__dirname`/`REPO_ROOT` constants (lines 13-15 of `generate-icons.mjs`).
5. Top-level config constants (ALL_CAPS), e.g. `WEBMANIFEST`, `SVGO_CONFIG`, `SIZE_BUDGET`.
6. Small helper functions above `main()` (e.g. `copyFile`, `writeBuffer`, `escapeXml`).
7. `async function main()` — top-level await via `main().catch(...)` tail.
8. Budget gates use `throw new Error(...)` or `process.exit(1)` — never silently continue.
9. Success-log lines: `console.log('  ' + verb + '  ' + path.relative(REPO_ROOT, dest) + '  (' + bytes + ' B)')`.
10. Error tail: `main().catch((err) => { console.error('[script-name] FAILED:', err); process.exit(1); })`.
11. No shebang line (`#!/usr/bin/env node`) — scripts invoked via `node scripts/X.mjs`, not executed directly.

**What NOT to do:**
- Don't use `commander` / `yargs` / `minimist` — project convention is zero arg-lib deps. Use `process.argv` slicing.
- Don't require transpilation (no TypeScript, no Babel) — `.mjs` + Node 22+ features are enough.
- Don't wire into `astro build` — scripts are explicitly author-invoked, output committed.
- Don't import from `src/` — scripts are self-contained; if shared logic is needed, extract to `scripts/_lib/*.mjs`.

### `node:test` + `node:assert/strict` Unit Test (mirror `tests/unit/shiki-palette-guard.test.ts`)

**Source:** `tests/unit/shiki-palette-guard.test.ts`, `tests/unit/rehype-code-badge.test.ts`, `tests/unit/vv-registry.test.ts`, `tests/unit/vv-geom.test.ts`, `tests/unit/vv-path.test.ts`.

**Apply to:** Every new unit test in Phase 4 (just the one: `tests/unit/excalidraw-to-svg.test.ts`).

**Contract:**
- Location: `tests/unit/*.test.ts`. Glob is wired in `package.json:11` — new files auto-discovered.
- Imports: `import { test } from 'node:test';` + `import assert from 'node:assert/strict';` as lines 1-2.
- Test shape: `test('descriptive name', () => { ... })` or `test('...', async () => { ... })` for async I/O.
- No test framework (Jest / Vitest / Mocha) — pure Node builtin runner.
- TypeScript: stripped by `--experimental-strip-types` flag, no compile step, no `tsconfig` for tests.
- Run: `npm run test:unit` (full suite, ~5s for ~35 existing tests + 8-9 new Phase 4 tests).
- Leading comment block (3-20 lines): What / Why / When to run / On failure. Mirror shiki-palette-guard's style — points at the code surface that must be updated if the test fails.
- Assertion messages hint at the failure mode: `assert.match(html, /color:#HEX/i, 'hex drifted — update global.css')`.
- For script integration tests, invoke the script via `spawnSync` from `node:child_process` — validates the CLI contract end-to-end including exit codes.
