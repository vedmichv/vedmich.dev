<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args → script | User (author) passes untrusted paths on command line |
| `.excalidraw.json` content → parser | JSON may contain arbitrary text in `elements[].text` (XSS-as-SVG risk in PLAN 02) |
| `files` blob → wrapper | Excalidraw's embedded raster images; oversized payload is a DoS vector |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Elevation of Privilege | CLI arg parsing (`srcPath`, `destPath`) | mitigate | PLAN 02 resolves both paths with `path.resolve()` + asserts result `startsWith(REPO_ROOT)`. **PLAN 01 contribution:** adds the failing test `::rejects-path-traversal` so the guard is testable before impl ships. |
| T-04-02 | Tampering / XSS-as-SVG | Text elements in `.excalidraw.json` | mitigate | Owned by PLAN 02 (SVGO `removeScripts` + `escapeXml()` on injected `<title>`/`<desc>`). PLAN 01 fixtures deliberately contain benign text only — no XSS payloads (test contract stays positive-path for this plan). |
| T-04-03 | Denial of Service | `files` blob size in JSON | mitigate | Owned by PLAN 02 (validate `files` blob total size < 100 KB). PLAN 01 fixtures omit the `files` key entirely (no embedded rasters in the reference diagrams). |

All three threats are surfaced in `04-RESEARCH.md §Security Domain`. Block severity: HIGH for T-04-01; MEDIUM for T-04-02/03.
</threat_model>

**Must deterministically trip the > 10 KB budget after SVGO multipass.**

**Strategy (belt-and-suspenders — avoids Issue #4 probabilistic-fixture risk):**
- **80 freedraw paths** with dense 20-point arrays (each freedraw point adds ~20 bytes of path data post-SVGO — doubling from the earlier 40-element draft).
- **PLUS 6 text elements spanning 3 distinct Excalidraw fonts** — Virgil (`fontFamily: 1`), Cascadia (`fontFamily: 3`), and Comic Shanns (`fontFamily: 4`). Each distinct font family triggers a separate `@font-face` embed (via `subset-font`) — roughly +800-1500 B per additional font subset even after aggressive subsetting. Three fonts together make the > 10 KB budget-trip deterministic regardless of freedraw-point compression.
