@shikijs/transformers v3 API (per RESEARCH.md Q1):
```typescript
import {
  transformerNotationHighlight,
  transformerNotationDiff,
} from '@shikijs/transformers';

// Each is a factory returning a ShikiTransformer object.
// Registered in shikiConfig.transformers: [transformerNotationHighlight(), transformerNotationDiff()]

// Classes emitted on output:
// - transformerNotationHighlight: adds `has-highlighted` to <pre>, `highlighted` to each `.line` <span> that had `// [!code highlight]`
// - transformerNotationDiff: adds `has-diff` to <pre>, `diff add` (two classes) or `diff remove` to each annotated `.line` <span>
```

Transformer annotation syntax per language (per RESEARCH.md Pitfall 6):
- YAML/bash: use `# [!code highlight]`, `# [!code ++]`, `# [!code --]` (line comment is `#`)
- TypeScript/JavaScript: use `// [!code highlight]`, `// [!code ++]`, `// [!code --]`
- Dockerfile: use `# [!code highlight]`, etc.
- The transformer prefix-matches the `[!code X]` literal regardless of comment char — use the correct language comment char on each fence.
